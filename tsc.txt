给我生成一个完整的桌面自动化软件项目，注意：不要给我生成网站，（非演示级，必须可运行且结构清晰），目标是构建一个“低token AI自动化执行系统”。
━━━━━━━━━━━━━━━━━━
【项目目标】
构建一个系统，使AI只需极少调用（1~3次），即可控制本地自动化执行复杂任务。
核心思想：
AI只负责规划（生成DSL）
本地系统负责执行全部逻辑
所有复杂能力封装为本地模块
最大限度减少token消耗
━━━━━━━━━━━━━━━━━━
【整体架构】
必须实现三层架构：
规划层（LLM接口）
接收用户自然语言
输出极简DSL或工具调用
不参与执行
执行层（核心引擎）
解析DSL
状态机驱动执行
支持循环、条件、异常处理
感知层（能力模块）
提供视觉识别、系统检测、交互能力
被执行层调用
━━━━━━━━━━━━━━━━━━
【技术要求】
前端：
Electron + React + TypeScript
三栏UI（节点库 / 画布 / 属性面板）
使用React Flow实现流程图编辑
支持拖拽、连线、缩放、节点编辑
后端执行：
Python
使用：
pyautogui（操作）
OpenCV（图像识别）
PaddleOCR或Tesseract（文字识别）
通信：
Electron 与 Python通过 IPC / HTTP通信
CLI：
提供命令行工具：
lobster run "任务"
━━━━━━━━━━━━━━━━━━
【DSL设计（必须实现）】
语法必须极简：
CLICK <target>
WAIT <condition>
LOOP <tag>
IF <condition>
END
RUN <macro>
要求：
每行一条指令
默认text匹配
支持嵌套结构
可解析为AST
━━━━━━━━━━━━━━━━━━
【执行引擎（核心）】
必须实现：
DSL解析器 → AST
状态机执行器
节点调度系统
事件回调（用于UI高亮）
支持：
循环执行
条件判断
子流程调用
暂停/恢复/停止
━━━━━━━━━━━━━━━━━━
【感知层模块（必须实现）】
图像识别
模板匹配（多尺度）
多目标检测
相似度阈值
OCR识别
文本提取
模糊匹配
区域分析
区域变化检测（帧差/SSIM）
区域稳定检测
颜色检测
进度条检测
自动识别方向
计算填充比例
场景识别
基于图像分类或特征判断
━━━━━━━━━━━━━━━━━━
【交互模块】
必须实现：
clickTarget（语义点击）
OCR + 图像 + 位置融合
waitUntil（统一等待）
支持多种条件
autoPopup（自动弹窗处理）
humanAction（模拟人类行为）
随机延迟
曲线鼠标移动
━━━━━━━━━━━━━━━━━━
【执行控制】
必须实现：
自动重试
超时机制
异常捕获
fallback策略（OCR失败→图像→位置）
━━━━━━━━━━━━━━━━━━
【高级模块（RUN宏）】
必须支持：
RUN 副本

要求：
RUN会展开为完整流程
支持用户自定义宏

【流程图系统】
必须支持：
节点拖拽

循环
子流程
节点复制粘贴
